# Design Token Transformation Instructions

## Task Definition
Transform the provided JSON file containing MUI design tokens into a W3C/Tokens Studio-compliant format following these exact specifications without deviation.

## Input and Output
- **Input**: Raw MUI token JSON file
- **Output**: W3C/Tokens Studio-compliant JSON file

## Transformation Rules (MUST FOLLOW EXACTLY)

### 1. Global Structure
- Create a top-level object with a single key: `"Primitives"`
- Place all primitive tokens as direct children of this object (no nested categories)
- Maintain the original MUI dot notation for token names (e.g., `"blue.500"`)

### 2. Token Properties
For EACH token, include EXACTLY these three properties:
- `$value`: The raw value (hex color, size in px/rem, etc.)
- `$type`: The W3C type identifier (e.g., "color", "dimension", "fontWeights")
- `$description`: Human-readable description derived from the token name

### 3. Token Name Derivation
- Preserve the original token path as the token name
- Use dot notation (e.g., `"blue.500"`, `"fontSize.sm"`)
- NEVER create your own naming scheme

### 4. Token Value Processing
- Do NOT transform or calculate the token values
- Copy the values exactly as they appear in the input
- For colors, maintain the exact same format (hex, rgb, etc.)

### 5. Type Determination
Determine `$type` by these exact rules:
- If value is a hex code or rgb/rgba format → `$type`: `"color"`
- If value has px, rem, em, % → `$type`: `"dimension"`
- If value is a number with no unit → check context:
  - If in a fontSize context → `$type`: `"dimension"`
  - If in a fontWeights context → `$type`: `"fontWeights"`
  - If in a letterSpacing context → `$type`: `"dimension"`
  - If in a lineHeight context → `$type`: `"dimension"` (even if unitless)
  - If in an opacity context → `$type`: `"number"`
- If value is a font name or comma-separated list → `$type`: `"fontFamilies"`

### 6. Description Formatting
- Format as lowercase space-separated words
- Replace dots with spaces
- Example: `"blue.500"` → `"blue 500"`
- Example: `"fontSize.sm"` → `"font size sm"`

## Example Transformation

INPUT:
```json
{
  "palette": {
    "blue": {
      "500": "#2196f3"
    },
    "fontSize": {
      "sm": "0.875rem"
    }
  }
}
```

OUTPUT:
```json
{
  "Primitives": {
    "blue.500": {
      "$value": "#2196f3",
      "$type": "color",
      "$description": "blue 500"
    },
    "fontSize.sm": {
      "$value": "0.875rem",
      "$type": "dimension",
      "$description": "font size sm"
    }
  }
}
```

## Critical Requirements
1. Do NOT add any properties beyond `$value`, `$type`, and `$description`
2. Do NOT nest tokens in categories - all tokens must be direct children of `Primitives`
3. Do NOT modify or manipulate the token values in any way
4. Do NOT create new token names - use the exact path from the input
5. Do NOT omit any tokens from the input
6. Include ALL tokens that have a value (skip objects that only contain other objects)

## Validation Checklist
Before submitting your result, verify that:
- [ ] All tokens are direct children of the `Primitives` object
- [ ] Every token has exactly the three required properties
- [ ] All token names use the original dot notation from input paths
- [ ] No values have been modified from the input
- [ ] Types have been correctly assigned according to the rules
- [ ] Descriptions are properly formatted
- [ ] The output is valid JSON

Process the ENTIRE input file, no matter how large, and maintain all token information from the original.
