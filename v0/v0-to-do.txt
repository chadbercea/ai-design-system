What you must tell Style Dictionary using sd-transforms:
	1.	Build a transform group that outputs:
	•	A CSS file containing a single :root block
	•	A scoped :root.dark block if you want to support dark mode
	2.	Emit only the required tokens listed above
	•	These can be mapped directly in a semantic alias layer inside Style Dictionary
	•	If your source tokens are named differently (colorPrimaryBase etc.), alias them into the required shadcn names
	3.	Apply an HSL triple transform to the color tokens
	•	Use or extend sd-transforms to emit 240 100% 50% instead of #3b82f6 or hsl(...)
	•	This must be done only for the shadcn-expectant color tokens
	4.	Write the result to build/css/variables.css
	•	Then import that file from app/globals.css in your Tailwind project

    :root {
  --background: 0 0% 100%;
  --foreground: 222.2 84% 4.9%;
  --primary: 240 100% 50%;
  --primary-foreground: 210 40% 98%;
  --radius: 0.5rem;
}

That’s all shadcn expects. No nesting. No var() chaining. No hex.
