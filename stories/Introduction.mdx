# Design System Pipeline Demo

## How It Works

This Storybook demonstrates a complete design token pipeline that enables **one design system to drive multiple frameworks** from a single source of truth.

### The Pipeline

```
1. Design in Figma
   ↓
2. Token Studio syncs → token-studio-sync-provider/DDS Foundations.json
   ↓
3. Run: npm run build:tokens
   ↓
4. Style Dictionary transforms tokens
   ↓
5. Generates framework-specific themes:
   • build/mui/theme.js
   • build/tailwind/theme.js  
   • build/shadcn/theme.js
   ↓
6. This Storybook imports those themes
   ↓
7. Components render with your design system
```

### What This Demonstrates

**The Problem:**
- Each framework (MUI, Tailwind, Shadcn) has different default styles
- Managing design consistency across frameworks is manual and error-prone
- Changes require updating multiple codebases

**The Solution:**
- One source of truth in Figma
- Automated transformation to all frameworks
- Change once, update everywhere

### Try It Yourself

1. Navigate to **"Comparison / All Frameworks"** - See the before/after
2. Check individual framework sections for detailed component examples
3. To see the live pipeline:
   - Change a color in Figma (e.g., Blue.500)
   - Token Studio syncs the change
   - Run `npm run build:tokens` in terminal
   - Refresh this Storybook
   - All DDS-themed components update automatically
